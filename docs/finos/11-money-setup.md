# EPIC-009 Money Setup

## Статус

Draft

## Цель

Собрать минимальные данные, чтобы FinOS мог честно рассчитывать безопасный дневной лимит.

Не делать сложную анкету.  
Не превращать старт в бухгалтерию.

Нужно мягко спросить только то, без чего нельзя считать безопасные траты.

## Проблема

Сейчас FinOS показывает:

Доступно сейчас

Но не может честно показать:

Безопасно сегодня

Потому что не знает:

- дату следующего дохода;
- примерный размер дохода;
- обязательные платежи;
- сколько дней нужно прожить до следующего дохода.

## Главная идея

Добавить короткую настройку финансовой базы.

Формат:

3–4 простых вопроса, которые можно пройти за 1 минуту.

## User Story

Как пользователь,  
я хочу один раз указать ключевые финансовые данные,  
чтобы FinOS мог подсказать, сколько безопасно тратить сегодня.

## Setup v1

Вопросы:

1. Когда следующий доход?
2. Сколько примерно придёт?
3. Какие обязательные платежи есть до этого дня?
4. Нужно ли учитывать семейный баланс?

## Scope v1

Входит:

- найти, какие данные уже есть в store;
- понять, можно ли использовать recurringTransactions;
- понять, куда сохранить nextIncomeDate / expectedIncomeAmount;
- предложить минимальный UX;
- не писать код до диагностики.

## Out of Scope

Не входит:

- полноценный бюджет;
- банковская интеграция;
- AI-прогноз;
- графики;
- подписки;
- сложный календарь;
- автоматический импорт.

## Принципы

1. Лучше 3 вопроса, чем 15 полей.
2. Пользователь не должен чувствовать себя бухгалтером.
3. Не показывать ложную точность.
4. Setup должен быть пропускаемым.
5. Можно заполнить позже.

## Возможный UX

На Today под “Доступно сейчас”:

Чтобы посчитать безопасный лимит, добавьте дату дохода и обязательные платежи.

Кнопка:

Настроить лимит

После нажатия:

1. Следующий доход
2. Сумма дохода
3. Обязательные платежи до дохода
