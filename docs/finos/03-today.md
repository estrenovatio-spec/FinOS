# Today

## Статус

Core Screen / v0.2 Perfect Morning

## Смысл экрана

Today — главный экран FinOS.

Это не лента, не дашборд и не бухгалтерская сводка.

Это утренний финансовый пульт человека.

Пользователь открывает FinOS и за 5 секунд понимает:

- сколько денег доступно;
- можно ли сегодня спокойно тратить;
- какой ближайший платёж;
- хватает ли до зарплаты;
- есть ли риск;
- какое одно действие сделать сейчас.

## Главный вопрос пользователя

> Что сейчас с моими деньгами?

## Главный принцип

На экране Today нельзя показывать всё.

Показываем только то, что помогает принять решение сегодня.

## Пользовательские сценарии

### Сценарий 1. Утро

Пользователь открывает FinOS утром.

Он должен сразу увидеть:

- можно потратить сегодня;
- баланс;
- дни до зарплаты;
- следующий платёж;
- короткий AI-инсайт;
- кнопку быстрого ввода.

### Сценарий 2. После покупки

Пользователь хочет быстро добавить расход.

Он должен сделать это за 5 секунд:

1. Открыл экран.
2. Нажал на поле или микрофон.
3. Ввёл: "500 продукты".
4. Операция добавлена.

### Сценарий 3. Перед тратой

Пользователь хочет понять, можно ли купить что-то сегодня.

Экран должен ответить:

- сколько можно спокойно потратить;
- не сломает ли это бюджет;
- есть ли ближайший платёж.

## Что показываем

### 1. Главный финансовый статус

Формулировка:

> Сегодня можно потратить

Пример:

> Сегодня можно потратить  
> 3 250 ₽

Это главная цифра экрана.

Она важнее общего баланса.

### 2. Баланс

Показываем коротко:

- общий баланс;
- личный баланс;
- баланс партнёра, если включён семейный режим.

Если семейный режим выключен, не показываем партнёра.

### 3. До зарплаты / следующего дохода

Пример:

> До зарплаты: 11 дней

Если дата дохода неизвестна:

> До следующего дохода дата не указана

И мягкая кнопка:

> Указать дату дохода

### 4. Следующий платёж

Показываем один ближайший платёж.

Пример:

> Следующий платёж  
> Интернет — 1 200 ₽ через 2 дня

Если платежей нет:

> Ближайших платежей нет

### 5. Подушка

Показываем компактно:

> Подушка  
> Защита на 1.3 месяца

Не показываем сложные расчёты на главном экране.

### 6. AI-инсайт дня

Один короткий инсайт.

Формула:

1. Факт.
2. Смысл.
3. Одно действие.

Пример:

> Расходы на продукты выше обычного на 18%.  
> Проверьте последние 3 покупки.

AI-инсайт не должен занимать больше 25% экрана.

### 7. Быстрый ввод

Главное действие:

> Добавить расход

Поддерживаем:

- текст;
- голос;
- быструю операцию.

Примеры ввода:

- 500 продукты
- 1200 такси
- пришло 150000 зарплата
- 3000 подарок Марфе

### 8. Последние операции

Показываем 3–5 последних операций.

Карточки компактные.

На карточке:

- категория;
- комментарий/описание;
- сумма;
- дата или время.

## Что скрываем

На экране Today НЕ показываем:

- курсы валют;
- длинные графики;
- услуги;
- страхование;
- инвестиционные детали;
- бизнес-аналитику;
- сложные фильтры;
- длинные объяснения;
- рекламу;
- все цели сразу;
- все долги сразу;
- все фонды сразу.

## Основное действие

Добавить операцию.

Кнопка или поле должны быть видны без скролла.

## Вторичные действия

- открыть все операции;
- открыть следующий платёж;
- открыть подушку;
- открыть AI-инсайт;
- открыть настройки Today.

## Empty State

Если данных мало, экран не должен выглядеть пустым.

Показываем:

> Добро пожаловать в FinOS.  
> Добавьте первую операцию — и я начну понимать ваши деньги.

Главная кнопка:

> Добавить первую операцию

Дополнительно:

> Указать зарплату  
> Создать первую цель

## Error State

Если данные не загрузились:

> Не удалось загрузить данные.  
> Попробуйте обновить экран.

Кнопка:

> Обновить

Если AI-инсайт недоступен:

> Сегодня без совета. Основные данные работают.

AI-ошибка не должна ломать главный экран.

## Визуальная иерархия

Порядок сверху вниз:

1. Приветствие.
2. Можно потратить сегодня.
3. Баланс / до зарплаты.
4. Следующий платёж.
5. Подушка.
6. AI-инсайт.
7. Быстрый ввод.
8. Последние операции.

## Язык интерфейса

Используем человеческие формулировки.

Плохо:

- свободные денежные средства;
- операционный остаток;
- регулярная транзакция;
- финансовая аналитика.

Хорошо:

- можно потратить;
- осталось до зарплаты;
- следующий платёж;
- совет дня;
- последние траты.

## Правила адаптации

### Новичок

Показываем минимум:

- можно потратить;
- баланс;
- быстрый ввод;
- последние операции.

### Семья

Добавляем:

- общий баланс;
- партнёр;
- семейные траты;
- общие цели.

### Продвинутый пользователь

Добавляем:

- подушка;
- долги;
- лимиты;
- фонды.

### Предприниматель

Не перегружаем Today бизнесом.

Показываем только один бизнес-инсайт, если он критичен:

> Сегодня ожидается поступление по проекту.

Подробности — в разделе Бизнес.

## Acceptance Criteria

- [ ] Пользователь понимает финансовое состояние за 5 секунд.
- [ ] Главная цифра экрана — «можно потратить сегодня».
- [ ] Быстрый ввод доступен без скролла.
- [ ] Добавление операции возможно менее чем за 5 секунд.
- [ ] На экране нет курсов валют.
- [ ] На экране нет услуг.
- [ ] На экране нет длинных текстов.
- [ ] AI-инсайт короткий и actionable.
- [ ] Если данных нет, есть понятный empty state.
- [ ] Если AI не работает, основной экран не ломается.
- [ ] Семейные блоки скрыты, если семейный режим выключен.
- [ ] Бизнес-блоки скрыты, если бизнес-режим выключен.

## Codex Notes

Codex должен сначала найти текущий главный экран приложения.

Нужно не добавлять новый хаос, а заменить/упростить текущий главный экран под концепцию Today.

Если текущая архитектура не позволяет сразу реализовать всё, делать поэтапно:

1. Убрать лишнее с главного экрана.
2. Сделать быстрый ввод центральным.
3. Добавить блок «можно потратить».
4. Добавить ближайший платёж.
5. Добавить короткий AI-инсайт.
6. Скрыть семейные/бизнес/валютные блоки по настройкам.

Не добавлять новые тяжёлые зависимости.

Не ломать текущие данные пользователя.
