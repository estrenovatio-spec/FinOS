# FinOS Docs

> Product context for Codex and future developers.

FinOS — Financial Operating System.

Это не просто приложение учёта расходов.  
Это финансовый помощник, который помогает человеку понимать деньги, принимать решения и двигаться к финансовому спокойствию.

## Главные продуктовые принципы

1. Один экран — одна задача.
2. Пользователь не должен чувствовать себя бухгалтером.
3. Быстрый ввод должен занимать 3–5 секунд.
4. AI и память должны помогать, а не читать лекции.
5. Сложность скрыта, польза видна.
6. Не добавлять функции ради функций.
7. Любое изменение должно снижать тревогу пользователя о деньгах.

## Что нельзя делать без отдельного RFC

- добавлять большие модули;
- менять модель данных;
- добавлять backend;
- добавлять подписки;
- менять архитектуру категорий;
- переписывать Today Screen с нуля;
- менять быстрый ввод без проверки сценариев;
- добавлять новые зависимости;
- делать “умный AI”, если можно решить простыми правилами.

## Текущий статус продукта

### EPIC-001 Perfect Morning

Статус: реализован первый Today Screen.

Сделано:

- главный экран стал Today;
- убраны курсы валют и лишняя аналитика;
- добавлен главный статус;
- быстрый ввод виден на главном экране;
- последние операции компактнее.

### EPIC-002 Fast Operation Input

Статус: реализовано ядро быстрого ввода.

Сделано:

- `500 продукты`;
- `кофе 300`;
- `такси 1200`;
- `пришло 100000 зарплата`;
- `1к кофе`;
- `2.5к продукты`;
- понятное подтверждение после добавления;
- correction memory после исправления категории.

## Обязательный формат работы Codex

Перед изменениями:

1. Найти релевантные документы.
2. Найти релевантные файлы кода.
3. Написать короткую диагностику.
4. Предложить план.
5. Вносить минимальные изменения.
6. Запустить `npm run build`.
7. Дать отчёт.

## Формат отчёта Codex

В конце каждой задачи писать:

- какие документы прочитаны;
- какие файлы найдены;
- что изменено;
- что специально не трогалось;
- как проверить вручную;
- какие риски остались;
- результат `npm run build`.

## Главный критерий качества

Если изменение не помогает человеку быстрее понять деньги или проще записать операцию — оно не нужно.
