# EPIC-002.1 Input Confirmation UX

## Статус

Draft

## Цель

После добавления операции FinOS должен коротко показать, как он понял ввод пользователя.

Это снижает тревогу, повышает доверие и делает быстрый ввод ощущением “магии”, а не чёрного ящика.

## Проблема

Сейчас пользователь пишет:

500 продукты

Операция добавляется, но пользователь не всегда понимает:

- какую сумму записали;
- в какую категорию попало;
- доход это или расход;
- можно ли быстро исправить ошибку.

## Главная идея

После успешного добавления показывать короткое подтверждение:

Добавлено: Продукты −500 ₽

или

Добавлено: Зарплата +100 000 ₽

Дополнительно дать действие:

Изменить

## User Story

Как человек, который быстро записывает расходы,  
я хочу сразу видеть, как FinOS понял мою фразу,  
чтобы быстро заметить ошибку и исправить её.

## UX-принципы

1. Подтверждение должно быть коротким.
2. Оно не должно мешать следующему вводу.
3. Оно не должно превращаться в модальное окно.
4. Ошибка должна исправляться быстро.
5. Пользователь должен чувствовать контроль.

## Формат подтверждения

### Расход

Добавлено: Продукты −500 ₽

### Доход

Добавлено: Зарплата +100 000 ₽

### Несколько операций

Добавлено 3 операции на сумму −2 400 ₽

### Неизвестная категория

Добавлено: Прочее −500 ₽  
Можно изменить категорию.

## Где показывать

Базовый вариант v1:

- toast после добавления операции.

Лучший вариант v2:

- компактная inline-карточка под быстрым вводом на 3–5 секунд;
- кнопка “Изменить”;
- возможность открыть редактирование последней операции.

## Scope v1

Входит:

- улучшить toast после успешного добавления;
- показывать категорию и сумму;
- различать доход и расход;
- для нескольких операций показывать количество;
- не ломать текущий список операций;
- не добавлять модальные окна, если их нет.

## Out of Scope

Не входит:

- полноценная correction memory;
- обучение AI на исправлениях;
- новый дизайн toast-системы;
- сложные анимации;
- новый backend;
- LLM.

## Acceptance Criteria

- [ ] После `500 продукты` появляется подтверждение: “Добавлено: Продукты −500 ₽”.
- [ ] После `кофе 300` появляется: “Добавлено: Кафе и рестораны −300 ₽”.
- [ ] После `пришло 100000 зарплата` появляется: “Добавлено: Зарплата +100 000 ₽”.
- [ ] После нескольких сумм появляется подтверждение количества операций.
- [ ] Пустой ввод не создаёт подтверждение.
- [ ] Если сумма не найдена, показывается понятная ошибка.
- [ ] Операция появляется в списке.
- [ ] Баланс пересчитывается.
- [ ] npm run build проходит.

## Implementation v0.1

## Статус

Implemented / Awaiting manual test

## Что реализовано

После быстрого добавления операции FinOS показывает более конкретное подтверждение:

- категория;
- знак операции;
- сумма.

Примеры:

- Добавлено: Продукты −500 ₽
- Добавлено: Кафе и рестораны −300 ₽
- Добавлено: Зарплата +100 000 ₽

## Почему это важно

Пользователь сразу видит, как FinOS понял его фразу.

Это повышает доверие к быстрому вводу и снижает тревогу ошибки.

## Проверка

- [ ] 500 продукты → Добавлено: Продукты −500 ₽
- [ ] кофе 300 → Добавлено: Кафе и рестораны −300 ₽
- [ ] такси 1200 → Добавлено: Транспорт −1 200 ₽
- [ ] пришло 100000 зарплата → Добавлено: Зарплата +100 000 ₽
- [ ] продукты → ошибка без создания операции
- [ ] операции появляются в списке
- [ ] баланс пересчитывается
- [ ] npm run build прошёл успешно
