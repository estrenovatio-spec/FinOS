# EPIC-002.2 Correction Memory

## Статус

Draft

## Цель

Сделать первую версию финансовой памяти FinOS.

Если пользователь исправил категорию операции, FinOS должен запомнить связь между фразой/словом и новой категорией.

## Главная идея

FinOS должен учиться на исправлениях пользователя.

Пример:

1. Пользователь вводит:
   пятёрочка 800

2. FinOS определяет:
   Продукты

3. Пользователь исправляет категорию:
   Покупки

4. FinOS запоминает:
   пятёрочка → Покупки

5. В следующий раз:
   пятёрочка 500

   FinOS выбирает:
   Покупки

## Почему это важно

Быстрый ввод должен становиться лучше со временем.

Если пользователь исправляет одно и то же несколько раз, а FinOS не учится — доверие падает.

## User Story

Как человек, который быстро записывает операции,  
я хочу, чтобы FinOS запоминал мои исправления,  
чтобы мне не приходилось повторять одно и то же.

## Scope v1

Входит:

- обнаружить, где пользователь редактирует операцию;
- определить, можно ли поймать изменение категории;
- сохранить простую связь keyword → categoryId;
- использовать эту связь перед fallback-категоризацией;
- не ломать существующие категории;
- не добавлять backend, если локального store достаточно.

## Out of Scope

Не входит:

- полноценный AI memory backend;
- сложные embeddings;
- облачная синхронизация памяти;
- обучение на всех полях;
- персональные LLM-модели;
- статистика уверенности;
- интерфейс управления памятью.

## Логика v1

### Что запоминать

Из исходной фразы операции выделять ключевые слова.

Примеры:

- пятёрочка 800 → пятёрочка
- кофе 300 → кофе
- авито 866 → авито
- школа Марфа 5000 → школа, марфа
- такси 1200 → такси

Если пользователь меняет категорию, сохранить:

keyword → categoryId

### Где использовать

При следующем быстром вводе:

1. Сначала planning input.
2. Потом correction memory.
3. Потом category keywords.
4. Потом fallback parser.
5. Потом Прочее.

## Пример

Память:

пятёрочка → groceries

Ввод:

пятёрочка 700

Результат:

categoryId = groceries

## Ограничения

- Не запоминать слишком общие слова:
  - купил
  - оплатил
  - пришло
  - получил
  - магазин
  - заказ
  - деньги
  - руб
  - рублей
- Не запоминать суммы.
- Не запоминать слова короче 3 символов.
- Не запоминать технические слова.

## Acceptance Criteria

- [ ] Если пользователь исправил категорию операции, FinOS может сохранить связь keyword → categoryId.
- [ ] При следующем вводе похожей фразы категория выбирается из памяти.
- [ ] Память имеет приоритет выше общего keyword fallback.
- [ ] Слишком общие слова не запоминаются.
- [ ] Суммы не запоминаются.
- [ ] Старый быстрый ввод не ломается.
- [ ] npm run build проходит.

## Проверочные сценарии

1. Ввести:
   пятёрочка 800

2. Исправить категорию на другую.

3. Ввести:
   пятёрочка 500

4. Проверить, что FinOS выбрал исправленную категорию.

## Implementation v0.1

## Статус

Implemented / Awaiting manual test

## Диагностика

Операция редактируется в:

- TransactionEditDialog.tsx

Категория сохраняется через:

- updateTransaction(...) в useStore.ts

Исходная фраза быстрого ввода сохраняется косвенно в:

- note

В проекте уже есть ai-memory:

- ai-memory.ts

## Решение

Не создавать новый механизм памяти.

Использовать существующий ai-memory и localStorage.

## Где хранится память

localStorage:

- voicebudget-ai-memory-v1

## Что изменено

В ai-memory.ts:

- усилена фильтрация общих слов;
- correction memory учится в первую очередь от исходной фразы before.note;
- сохранение correction memory происходит при изменении категории;
- добавлен тихий console.info только в development.

## Как используется память

При новом быстром вводе:

1. detectCategoryId(...) сначала проверяет matchAiMemoryCategoryId(...).
2. Если есть совпадение из correction memory, используется сохранённая категория.
3. Только потом идут обычные keywords и fallback.

## Что запоминается

- очищенные ключевые слова;
- короткие фразы;
- новая связка keyword / phrase → categoryId.

## Что не запоминается

- суммы;
- короткие слова;
- шумовые стоп-слова;
- магазин;
- заказ;
- деньги;
- карта;
- пришло;
- получил;
- руб;
- рублей.

## Изменённые файлы

- ai-memory.ts

## Риски

- память локальная и не синхронизируется между устройствами;
- если note сильно отредактирован, обучение может быть слабее;
- короткие фразы могут иногда матчиться шире, чем одно ключевое слово.

## Проверка

- [ ] пятёрочка 800 → исправить категорию → пятёрочка 500 использует исправленную категорию.
- [ ] авито 866 → исправить категорию → авито 300 использует исправленную категорию.
- [ ] обычные фразы быстрого ввода продолжают работать.
- [ ] npm run build прошёл успешно.
