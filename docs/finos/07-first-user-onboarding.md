# EPIC-004 First User Onboarding

## Статус

Draft

## Цель

Сделать первое открытие FinOS понятным, спокойным и быстрым.

Новый пользователь должен сразу понять:

- что это не бухгалтерия;
- что первое действие — добавить операцию;
- что можно писать обычным языком;
- что FinOS начнёт помогать после первых данных.

## Проблема

Если у пользователя нет операций, целей, подушки и истории, экран Today может выглядеть пустым или странным.

Например:

- “Сегодня можно потратить 0 ₽”
- баланс 0 ₽
- нет операций
- нет совета

Для нового пользователя это может выглядеть как бесполезный экран.

## Главная идея

Пустое состояние Today должно быть не пустотой, а мягким стартом.

Формула:

1. Коротко объяснить ценность.
2. Дать один пример.
3. Попросить добавить первую операцию.
4. Не показывать сложность.

## User Story

Как новый пользователь,  
я хочу сразу понять, что мне делать в FinOS,  
чтобы без обучения начать вести деньги.

## Первый экран для нового пользователя

Если нет операций, показываем:

Добро пожаловать в FinOS

Начните с одной операции:

500 продукты

После этого я покажу, что происходит с деньгами.

Главное действие:

Добавить первую операцию

## Что НЕ показывать новичку

Если операций нет, не нужно акцентировать:

- 0 ₽ можно потратить;
- пустой баланс;
- пустую подушку;
- пустые долги;
- пустую аналитику;
- сложный AI;
- услуги;
- бизнес;
- инвестиции.

## Onboarding v1

Не делать многоэкранный onboarding.

Нужен только умный empty state на Today.

## Scope v1

Входит:

- определить, что пользователь новый;
- если операций нет, показать welcome-state;
- сделать быстрый ввод главным действием;
- показать 2–3 примера фраз;
- не ломать обычный Today, если операции уже есть.

## Out of Scope

Не входит:

- регистрация;
- аккаунты;
- пошаговый тур на 10 экранов;
- прогресс-бар обучения;
- видео;
- платная подписка;
- сложная геймификация;
- отдельный backend.

## Условие нового пользователя

Пользователь считается новым, если:

- нет операций;

или, если технически доступно:

- операций меньше 3.

Для v1 достаточно условия:

transactions.length === 0

## Тексты

### Заголовок

Добро пожаловать в FinOS

### Подзаголовок

Начните с одной операции — и я покажу, что происходит с деньгами.

### Примеры

- 500 продукты
- кофе 300
- пришло 100000 зарплата

### Подсказка

Пишите обычным языком. Категорию выберу сам.

## Acceptance Criteria

- [ ] Если операций нет, Today показывает welcome-state.
- [ ] На welcome-state не акцентируется “можно потратить 0 ₽”.
- [ ] Быстрый ввод виден сразу.
- [ ] Есть примеры фраз.
- [ ] После добавления первой операции Today переключается в обычный режим.
- [ ] Старый Today для пользователя с операциями не ломается.
- [ ] Быстрый ввод работает.
- [ ] npm run build проходит.

## Проверка

1. Очистить операции или открыть новый профиль.
2. Увидеть welcome-state.
3. Ввести:
   500 продукты
4. Проверить, что операция добавилась.
5. Проверить, что экран стал обычным Today.
