# EPIC-005 First 7 Days / Habit Loop

## Статус

Draft

## Цель

Сформировать у нового пользователя привычку ежедневно записывать операции без ощущения бухгалтерии.

FinOS должен мягко помогать человеку возвращаться каждый день.

## Проблема

Пользователь может добавить первую операцию, увидеть, что всё работает, и больше не вернуться.

Для финансового продукта важно не только первое действие, но и первые 7 дней.

## Главная идея

Первые дни FinOS должен давать простую ежедневную миссию:

Запишите 3 операции сегодня.

Это достаточно маленькое действие, чтобы не пугать пользователя, но достаточно полезное, чтобы начать формировать данные.

## User Story

Как новый пользователь,  
я хочу понимать, что мне делать сегодня,  
чтобы постепенно привыкнуть вести деньги.

## Mission v1

### Если сегодня записано 0 операций

Миссия дня: запишите 3 операции  
Начните с любой траты: 500 продукты

### Если сегодня записана 1 операция

Миссия дня: запишите 3 операции  
Осталось: 2

### Если сегодня записаны 2 операции

Миссия дня: запишите 3 операции  
Осталось: 1

### Если сегодня записаны 3+ операции

Готово на сегодня ✅  
Вы записали 3 операции. Этого достаточно, чтобы FinOS начал понимать ваши деньги.

## Где показывать

На Today, ниже быстрого ввода или рядом с советом дня.

Не выше главного финансового статуса для обычного пользователя.

Для нового пользователя welcome-state важнее mission card.

## Scope v1

Входит:

- посчитать количество операций за сегодня;
- показать компактную mission card;
- обновлять её после добавления операции;
- не ломать Today;
- не добавлять backend;
- не добавлять геймификацию.

## Out of Scope

Не входит:

- streaks;
- уровни;
- награды;
- push-уведомления;
- email;
- календарь привычек;
- сложная аналитика удержания;
- платная подписка.

## UX-принципы

1. Миссия должна быть спокойной.
2. Не давить на пользователя.
3. Не использовать токсичную мотивацию.
4. Не превращать FinOS в игру ради игры.
5. Главная ценность — финансовое спокойствие.

## Acceptance Criteria

- [ ] Today считает операции за сегодня.
- [ ] Если сегодня 0 операций, показывает миссию на 3 операции.
- [ ] Если сегодня 1 операция, показывает “Осталось: 2”.
- [ ] Если сегодня 2 операции, показывает “Осталось: 1”.
- [ ] Если сегодня 3+ операции, показывает “Готово на сегодня”.
- [ ] После добавления операции миссия обновляется.
- [ ] Новый welcome-state не ломается.
- [ ] Обычный Today не перегружен.
- [ ] npm run build проходит.

## Проверка

1. Очистить операции.
2. Добавить первую операцию.
3. Проверить mission card.
4. Добавить вторую.
5. Проверить “Осталось: 1”.
6. Добавить третью.
7. Проверить “Готово на сегодня”.
