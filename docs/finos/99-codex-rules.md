# Codex Rules

## Главный принцип

Codex не должен добавлять функции без документации.

## Перед изменением кода

Codex обязан:

1. Найти связанный документ.
2. Понять цель изменения.
3. Сформулировать план.
4. Изменить минимально необходимое количество файлов.
5. Проверить сборку.

## Запрещено

- добавлять большие модули без RFC;
- переписывать архитектуру без причины;
- ломать существующие пользовательские данные;
- усложнять UX;
- добавлять новые зависимости без необходимости.

## Формат ответа Codex

В конце задачи Codex должен написать:

- что изменено;
- какие файлы затронуты;
- как проверить;
- какие риски остались.
