# EPIC-006 First Day Insight

## Статус

Draft

## Цель

Дать пользователю первую понятную пользу после того, как он записал несколько операций за день.

FinOS должен не только просить записывать операции, но и показывать, что он уже начал понимать деньги пользователя.

## Проблема

После 3 операций пользователь видит, что привычка выполнена, но ещё не получает явной аналитической пользы.

Нужно показать маленький инсайт, чтобы у пользователя появилось ощущение:

“О, это действительно помогает.”

## Главная идея

Когда за сегодня записано 3+ операции, показать короткий инсайт дня.

Пример:

Сегодня вы записали 3 операции.  
Больше всего ушло на транспорт — 1 200 ₽.

## User Story

Как пользователь, который записал несколько операций,  
я хочу увидеть простой вывод по дню,  
чтобы понять, зачем я это делаю.

## Scope v1

Входит:

- посчитать операции за сегодня;
- посчитать расходы за сегодня;
- найти категорию с максимальной суммой расходов;
- показать короткий инсайт;
- не использовать LLM;
- не добавлять backend;
- не добавлять сложную аналитику.

## Out of Scope

Не входит:

- недельный AI-разбор;
- месячный отчёт;
- графики;
- прогнозы;
- советы по экономии;
- LLM;
- пуши;
- email;
- сравнение с прошлым месяцем.

## Логика v1

Если сегодня меньше 3 операций:

- не показывать First Day Insight.

Если сегодня 3+ операции и есть расходы:

Показать:

Сегодня вы записали N операции.  
Больше всего ушло на {категория} — {сумма} ₽.

Если сегодня 3+ операции, но расходов нет, только доходы:

Показать:

Сегодня вы записали N операции.  
Доходы за день: {сумма} ₽.

Если есть и доходы, и расходы:

Можно показать:

Сегодня вы записали N операции.  
Расходы за день: {сумма} ₽. Больше всего — {категория}.

## Где показывать

На Today.

Лучшее место v1:

- после mission card;
- перед “Совет дня”;
- компактной карточкой.

## UX-принципы

1. Инсайт короткий.
2. Без морализаторства.
3. Без тревожных формулировок.
4. Не говорить “вы слишком много потратили”.
5. Не делать вид, что FinOS знает больше, чем знает.
6. Только факты по сегодняшним операциям.

## Acceptance Criteria

- [ ] Если сегодня меньше 3 операций, First Day Insight не показывается.
- [ ] Если сегодня 3+ операции, карточка появляется.
- [ ] Карточка показывает количество операций за сегодня.
- [ ] Карточка показывает категорию с максимальными расходами.
- [ ] Сумма форматируется красиво.
- [ ] Логика считает по полному списку сегодняшних операций, а не по preview.
- [ ] Onboarding не ломается.
- [ ] Mission card не ломается.
- [ ] Быстрый ввод не ломается.
- [ ] npm run build проходит.

## Проверка

1. Новый пользователь:
   - insight не показывается.

2. Добавить:
   - 500 продукты
   - кофе 300

   Ожидание:
   - insight ещё не показывается.

3. Добавить:
   - такси 1200

   Ожидание:
   - появляется insight:
     Сегодня вы записали 3 операции.
     Больше всего ушло на транспорт — 1 200 ₽.
