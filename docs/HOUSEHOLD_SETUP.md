# Общий бюджет: что делать по шагам

## Сначала решите, как вы ведёте бюджет

| Вариант | Когда | Что делать |
|---------|--------|------------|
| **Только на одном телефоне** | Ведёте сами, партнёр не пользуется приложением | Облако **не обязательно**. В настройках укажите **имя партнёра** — при записи выбирайте «я» или партнёра. |
| **Веду один, но хочу бэкап в облаке** | Один Telegram, данные не потерять | Режим **«Веду один»** в блоке «Облако и семья». |
| **Два телефона, один бюджет** | Оба пользуются VoiceBudget в Telegram | Режим **«Вдвоём»** + код приглашения. |

---

## Если нужно облако (два телефона или бэкап)

### Шаг 1. База данных (один раз)

1. Зарегистрируйтесь в [Supabase](https://supabase.com) или [Neon](https://neon.tech) (есть бесплатный тариф).
2. Создайте проект → **Project Settings → Database → Connection string**.

**Важно для Vercel:** возьмите **Transaction pooler** (хост `*.pooler.supabase.com`, порт **6543**), в конце добавьте `?pgbouncer=true`.  
Строка вида `...@db.xxx.supabase.co:5432` (direct) с Vercel часто **не подключается** — из‑за этого «Не удалось выполнить действие в облаке».

Для `npm run db:push` на Mac используйте **Direct connection** (порт 5432) — отдельно в `.env.local`, только на время создания таблиц.

### Шаг 2. Переменные на Vercel (один раз)

В проекте **voicebudget** на [vercel.com](https://vercel.com) → **Settings → Environment Variables**:

| Переменная | Откуда взять |
|------------|----------------|
| `DATABASE_URL` | Строка подключения из шага 1 |
| `TELEGRAM_BOT_TOKEN` | [@BotFather](https://t.me/BotFather) → ваш бот, который открывает Mini App |
| `HOUSEHOLD_SESSION_SECRET` | Любая длинная случайная строка (32+ символа) |

Сохраните и сделайте **Redeploy** (Deployments → … → Redeploy).

### Шаг 3. Таблицы в БД (один раз)

На своём компьютере в папке `voicebudget`:

**Вариант A — Supabase SQL Editor** (если `npm run db:push` зависает):

1. Supabase → **SQL Editor** → **New query**
2. Откройте файл `voicebudget/prisma/setup-tables.sql`, скопируйте **весь** текст
3. Вставьте в редактор → **Run**
4. Должно быть `Success` без ошибок

**Вариант B — терминал:**

```bash
cp .env.local.example .env.local
# в .env.local — Session pooler из Supabase (не host:5432 из примера!)

npm install
npm run db:push
```

Команда `npm run db:push` читает **`.env.local`** (обычный `npx prisma db push` его не видит).

Проверка: в `.env.local` не должно быть `@host:5432` — это шаблон из примера.

После этого таблицы созданы.

### Шаг 4. В Telegram — создать или войти в семью

1. Откройте Mini App **из Telegram** (не просто браузер).
2. **Настройки** (шестерёнка) → **«Облако и семья»**.

**Вариант А — ведёте один:**

- «Веду один» → имя партнёра (для меток) → **«Создать облачный бюджет»**.
- Если уже есть записи только на этом телефоне → **«Загрузить данные с этого телефона»** (один раз).

**Вариант Б — вдвоём:**

1. Тот, кто создаёт: «Вдвоём» → **«Создать облачный бюджет»** → запомните **код** (6 букв/цифр).
2. Второй человек: тот же бот → Настройки → код → **«Присоединиться»**.
3. Оба при необходимости нажимают **«Синхронизировать»**.

### Шаг 5. Тот же Telegram в браузере (компьютер)

1. Откройте https://voicebudget.vercel.app (не «просто сайт» без входа).
2. **Настройки → Облако и семья** → **«Войти через Telegram»** — тот же аккаунт, что в боте на телефоне.
3. Семью **создавать второй раз не нужно** — нажмите **«Подключить этот браузер»** (лимита устройств нет).
4. Сообщение «уже в семейном бюджете» значит, что облако на телефоне есть — нужно **подключить браузер**, а не создавать семью.
5. Если записи с телефона не видны: на главной **«Скачать с облака»** (или подождите ~30 сек).

После полной очистки базы на телефоне заново **«Создать облачный бюджет»**, затем в браузере снова **войти через Telegram**.

### Шаг 6. Как пользоваться каждый день

- Новые операции (голос/текст) сами уходят в облако, если оно подключено.
- Фильтры **Общий / Я / имя партнёра** — как раньше.
- Если партнёр добавил трату, а у вас не видно — **«Скачать с облака»** на главной (или откройте вкладку заново).

---

## Если облако не настраивать

Работает как раньше: всё в памяти телефона. Укажите имя партнёра в настройках — этого достаточно для учёта «кто потратил».

В блоке «Облако и семья» будет текст про отсутствие `DATABASE_URL` на сервере — это нормально, локальный режим не ломается.

## Частые вопросы

**«Сервер без БД» в настройках**  
На Vercel не задан `DATABASE_URL` или не был redeploy после добавления.

**Партнёр не видит мои траты**  
Он должен **присоединиться по коду** и открывать приложение **в Telegram**. Потом «Синхронизировать».

**Два разных бюджета на двух телефонах**  
Скорее всего облако не подключено — каждый копит данные локально. Создайте семью и присоединитесь по коду.

**На телефоне есть данные, в браузере пусто**  
В браузере не нажали **«Войти через Telegram»** (тот же аккаунт) или осталась старая сессия после очистки БД — войдите снова и нажмите **«Скачать с облака»**. Телефон не «главный»: облако одно, оба устройства должны быть в одной семье под одним Telegram.
